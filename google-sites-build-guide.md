# Google Sites Build Guide

This draft is designed for a simple academic personal website inspired by the earlier reference sites and the Bryan James lab site:

- `jilian0717.github.io`: concise academic profile, direct navigation, compact research/paper links.
- `yangyu-climate.github.io/#contact`: fuller one-page research profile with sections for About, Research, Publications, CV, and Contact.
- `bryandjames.com`: lab-style site with Latest News, Research, Publications, Our Team, and Openings.

Because this is currently an individual researcher website rather than a lab website, this draft borrows the clean news/research/publications rhythm but intentionally omits `Our Team` and `Openings`.

Google Sites does not import a custom HTML/CSS website as a full editable site, so use the included `index.html` as a visual mockup and rebuild the sections with Google Sites blocks.

## Recommended Site Settings

Site name: `Yanchen Sun`

Page type: Multi-page personal website with top navigation. Use one page each
for About, News, Research, Publications, Teaching & Mentoring, Service,
Mentoring & Outreach, and Contact.

Theme:

- Font style: Clean / Modern
- Header type: Cover or Large banner
- Color accents: deep teal, forest green, warm gold
- Navigation: Top navigation
- Footer: Simple copyright and last-updated line

## Pages

1. Home / Header
   - Use a full-width banner.
   - Add `Yanchen Sun, Ph.D.` as the main title.
   - Add `Research Associate | Woods Hole Oceanographic Institution` as the subtitle.
   - Add the research-vision statement and smaller current-research line from `site-copy-deck.md`.
   - Below the About introduction, add compact previews for Research Overview and Recent News.
   - Use the provided `assets/research-hero.png` as the header image, or replace it with your own research image or portrait.

2. About
   - Add a two-column section.
   - Left: section heading, `About`.
   - Right: use the three-paragraph research statement from `site-copy-deck.md`.
   - Keep the long-term independent-research-program paragraph in this section; it gives the site a clear faculty-facing narrative.

3. News
   - Add a compact reverse-chronological bullet feed inspired by the lab reference site.
   - Begin each entry with the year, followed by a short title and one-line context.
   - Place the full oral and poster presentation record below the updates using the same date-first bullet format.
   - Do not add lab news, team updates, student awards, or openings until you have an active lab group.

4. Research
   - Use the five research-theme sections from `site-copy-deck.md`, one project per row with a figure and accompanying description.
   - Add the Future Research Program and Collaborative Translation sections before the research themes.
   - Keep figure backgrounds white or transparent and use a consistent aspect ratio.

5. Publications
   - Add all papers from `site-copy-deck.md`.
   - Use grouped subheadings:
     - Peer-Reviewed Papers
     - Manuscripts in Review
     - Manuscripts in Preparation
   - Google Sites will be easier to edit if each paper title is bolded, with the citation line below it in normal text.
   - Link each available DOI using the DOI URL supplied in `site-copy-deck.md`.
   - Add a Google Scholar button above or below the list.

6. Teaching & Mentoring
   - Add the teaching approach, courses prepared to teach, mentoring programs, and teaching-preparation credentials from `site-copy-deck.md`.

7. Service and Professional Engagement
   - Use separate sections for academic service, outreach, and professional affiliations.
   - Include peer review, editorial/conference service, conference judging, and outreach.

8. Contact
   - Add email, office/lab, institution, location, and profile links.
   - Use one CV button/link labeled `CV`, linked to `assets/CV.pdf`.
   - Add profile buttons:
     - Google Scholar: `https://scholar.google.com/citations?user=2PysLtIAAAAJ`
     - LinkedIn: `https://www.linkedin.com/in/yanchensun`
   - If you do not want your email scraped, use an image or write it as `name [at] institution [dot] edu`.

## Google Sites Navigation

Use separate top navigation pages to match the draft:

- Home
- About
- News
- Research
- Publications
- Teaching & Mentoring
- Service and Professional Engagement
- Contact

Put publications, presentations, talks, awards, and other updates together on the News page as one reverse-chronological feed.

## Assets

- Hero/banner image: `assets/research-hero.png`
- CV: `assets/CV.pdf`
- Replace with your own portrait or research figure when available.
- If using a portrait, place it in the About section rather than the hero unless the portrait is high quality.

## Launch Checklist

- Replace any remaining placeholder text after you add profile links.
- Upload `CV.pdf` to Google Drive and set sharing to public/viewable.
- Link Google Scholar, LinkedIn, and email.
- Check mobile preview in Google Sites.
- Publish with a short URL slug, such as `sites.google.com/view/your-name`.
- Add a last-updated date in the footer.
